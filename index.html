CMAsnap Integration Guide

This documentation covers authentication strategies and embedding options for integrating CMAsnap into your applications.

Overview

CMAsnap provides multiple methods for authentication and integration:

  • OAuth2 with Auth0: Primary authentication with email/password or social providers
  • Tangilla SSO: Seamless single sign-on for Tangilla users
  • Embedding: Iframe integration for seamless CMAsnap functionality in your website

Quick Start

Tangilla SSO

Direct users to: https://app.cmasnap.com/tangilla-sso?mlsId=PROPERTY_ID

Embedding

<iframe
  id="cmasnap-iframe"
  src="https://app.cmasnap.com/new?embed=true&mlsId=PROPERTY_ID"
  scrolling="no"
  style="width: 1px; min-width: 100%; border: 0; overflow: hidden; height: 500px">
</iframe>

<script src="https://app.cmasnap.com/static/js/embed.js"></script>

<script>
  const cmasnapInt = setInterval(() => {
    if (typeof CMAsnap !== 'undefined') {
      new CMAsnap('#cmasnap-iframe');
      clearInterval(cmasnapInt);
    }
  }, 1000);
</script>

Authentication

Authentication strategies and SSO options

Tangilla Single Sign-On

Automatically authenticates users who are logged into Tangilla, providing seamless access to CMAsnap.

How it works:

  1. User clicks a link to https://app.cmasnap.com/tangilla-sso
  2. If user is logged into Tangilla, they're automatically authenticated
  3. User is redirected to the requested resource (e.g., property report)

Benefits:

  • No additional login required
  • Seamless user experience
  • Maintains context (mlsId, return URLs)

Example Implementation:

// Generate a Tangilla SSO link
function createTangillaLink(mlsId) {
  return `https://app.cmasnap.com/tangilla-sso?mlsId=${mlsId}`;
}

// Redirect to Tangilla SSO
window.location.href = createTangillaLink('PROP123456');
query Parameters
mlsId
string
Example: mlsId=PROP123456

MLS property ID - loads existing report or creates new one after authentication

returnTo
string
Example: returnTo=/reports/custom-view

Custom URL to redirect to after authentication

Responses

Production server

https://app.cmasnap.com/tangilla-sso

Auto-login with provider

Generic auto-login endpoint that attempts authentication with a specified provider.

Currently supports:

  • tangilla: Tangilla SSO provider

This endpoint is useful when you want to conditionally trigger auto-login based on user context.

query Parameters
returnTo
string
Example: returnTo=/new?mlsId=PROP789

URL to return to after successful authentication

Responses

Production server

https://app.cmasnap.com/auto-login

Create new CMA report with authentication options

Main entry point for creating CMA reports. Supports multiple authentication and access methods.

Note: When using mlsId or address parameters, the system will first attempt to find an existing report for that property. If found, it will load the existing report; otherwise, it will create a new one.

Authentication Methods:

1. Standard Authentication

Users not logged in are redirected to Auth0 login page.

2. Tangilla Auto-Login

Add ?autoLogin=tangilla to attempt automatic Tangilla authentication:

https://app.cmasnap.com/new?mlsId=PROP123&autoLogin=tangilla

3. Embed Mode

Add ?embed=true for iframe integration:

<iframe
  id="cmasnap-iframe"
  src="https://app.cmasnap.com/new?embed=true&mlsId=PROP123"
  scrolling="no"
  style="width: 1px; min-width: 100%; border: 0; overflow: hidden; height: 500px">
</iframe>

Combined Example:

<!-- Embedded CMAsnap with Tangilla auto-login -->
<iframe
  id="cmasnap-iframe"
  src="https://app.cmasnap.com/new?embed=true&autoLogin=tangilla&mlsId=PROP123"
  scrolling="no"
  style="width: 1px; min-width: 100%; border: 0; overflow: hidden; height: 500px">
</iframe>

<script src="https://app.cmasnap.com/static/js/embed.js"></script>

<script>
  const cmasnapInt = setInterval(() => {
    if (typeof CMAsnap !== 'undefined') {
      new CMAsnap('#cmasnap-iframe');
      clearInterval(cmasnapInt);
    }
  }, 1000);
</script>
query Parameters
mlsId
string
Example: mlsId=PROP123456

MLS listing ID - searches for existing report or creates new one for this property

address
string
Example: address=123%20Main%20St%2C%20Austin%2C%20TX

URL-encoded property address - searches for existing report or creates new one at this address

embed
boolean
Default: false

Enable embed mode for iframe usage

autoLogin
string
Value: "tangilla"

Attempt auto-login with specified provider (currently only 'tangilla')

Responses

Production server

https://app.cmasnap.com/new
